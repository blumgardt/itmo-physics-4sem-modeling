# Физика и код: магнитное поле соленоида

## Что моделируется

**Соленоид** — это катушка: провод намотан витками вдоль оси. Когда по нему течёт ток, появляется магнитное поле. Программа показывает это поле визуально.

---

## Шаг 1. Закон Био–Савара — "откуда берётся поле"

**Физика:** Любой кусочек провода с током создаёт вокруг себя маленькое магнитное поле. Закон Био–Савара говорит: насколько сильное — зависит от тока `I`, длины кусочка `dl` и расстояния до точки наблюдения `r`.

```
dB = (μ0 / 4π) · (I · dl × r) / r³
```

**В коде:** константа `mu0` — это физическая константа из закона Био–Савара, `μ₀ = 4π·10⁻⁷`:

```go
// main.go:19
mu0 = 4 * math.Pi * 1e-7
```

---

## Шаг 2. Эллиптические интегралы — "точная формула для одного витка"

**Физика:** Если взять один круглый виток с током и захотеть точно посчитать поле в произвольной точке рядом — получается сложная формула через специальные функции: **эллиптические интегралы K(m) и E(m)**. Это не страшно — просто такой математический инструмент для описания "несимметричных" ситуаций (точка не на оси витка).

**В коде:** функции `ellipticK` и `ellipticE` считают эти интегралы методом AGM (среднее арифметико-геометрическое) — быстрый итеративный способ (`main.go:29–67`):

```go
func ellipticK(m float64) float64 {
    a, g := 1.0, math.Sqrt(1-m)
    for math.Abs(a-g) > 1e-15 {
        a, g = (a+g)/2, math.Sqrt(a*g)  // усредняем пока не сойдётся
    }
    return math.Pi / (2 * a)
}
```

---

## Шаг 3. Поле одного витка — функция `singleLoopField`

**Физика:** Один круглый виток радиуса `R` с током `I` создаёт поле с двумя компонентами:
- `Bz` — вдоль оси катушки
- `Br` — поперёк оси (радиальная)

В центре на оси `Br = 0`, поле чисто осевое. Чем дальше от оси — тем сложнее картина.

**В коде:** `main.go:73–104`. Особый случай — точка прямо на оси (`ri < 1e-12`), там формула проще:

```go
// поле на оси витка (упрощённая формула)
denom := math.Pow(R*R + zi*zi, 1.5)
Bz[i] = mu0 * current * R*R / (2 * denom)
```

Для всех остальных точек — формулы через `K` и `E`:

```go
// общая точка, через эллиптические интегралы
Bz[i] = c / alpha * (K + (R*R - ri*ri - zi*zi) / beta2 * E)
Br[i] = c * zi / (ri * alpha) * (-K + (R*R + ri*ri + zi*zi) / beta2 * E)
```

---

## Шаг 4. Принцип суперпозиции — функция `solenoidField`

**Физика:** Соленоид = `N` витков. Каждый виток создаёт своё поле. **Полное поле = сумма полей всех витков.** Это и есть принцип суперпозиции:

```
B = B₁ + B₂ + B₃ + ... + Bₙ
```

**В коде:** `main.go:122–174`. Сначала витки расставляются равномерно вдоль длины `L`:

```go
// координаты центров витков
pitch := L / float64(N)
centers[i] = -L/2 + pitch*(float64(i)+0.5)
```

Потом для каждого витка считается поле и **прибавляется** к общему:

```go
// суперпозиция — суммируем вклад каждого витка
dBr, dBz := singleLoopField(R, current, rGrid, zShifted)
for j := 0; j < n; j++ {
    localBr[j] += dBr[j]
    localBz[j] += dBz[j]
}
```

Для скорости вычисления распараллелены на 8 горутин (`nWorkers = 8`).

## Шаг 5. Расчётная сетка — "в каких точках считаем"

**Физика:** Нельзя посчитать поле везде — поэтому берётся сетка точек (161×101 = ~16 000 точек) и в каждой считается поле.

**В коде:** `main.go:415–431`

```go
gridNZ = 161  // точек по оси z
gridNY = 101  // точек по оси y

zLin := linspace(-g.extZ, g.extZ, nz)
yLin := linspace(-g.extY, g.extY, ny)
```

## Шаг 6. Тепловая карта — "цвет = сила поля"

**Физика:** Чем ярче цвет — тем сильнее магнитное поле `|B|` в этой точке. Темнее = слабее. Хорошо видно, что внутри соленоида поле гораздо сильнее и однороднее.

Модуль поля вычисляется как:
```
|B| = √(Bz² + Br²)
```

**В коде:** `main.go:482–497` — для каждой точки сетки `|B|` нормируется и переводится в цвет по шкале plasma (тёмно-фиолетовый → жёлтый):

```go
t := bMag2D[j][i] / g.heatmapMax   // нормировка 0..1
r, g, b := plasmaColor(t)           // перевод в цвет
```

## Шаг 7. Силовые линии — метод Рунге–Кутта 2-го порядка

**Физика:** Силовые линии показывают **направление** поля. Это как нарисовать стрелки куда "смотрит" поле в каждой точке. Внутри соленоида они идут прямо вдоль оси, снаружи замыкаются вокруг катушки.

**Алгоритм:** Начинаем из точки, делаем маленький шаг вдоль вектора поля, повторяем. Метод RK2 (Рунге–Кутта 2-го порядка) делает это точнее чем простой шаг Эйлера — сначала пробный шаг на половину, потом полный шаг по полю из середины:

```go
// main.go:245–258 — один шаг RK2
dzStep := dir * bz1/mag * dt                             // пробный полушаг
bz2, by2, ok := interp(z+dzStep*0.5, y+dyStep*0.5)      // поле в середине
nextZ := z + dir*bz2/mag2*dt                             // настоящий шаг
```

Стартовые точки выбраны в разных местах — внутри, снаружи, у краёв — чтобы нарисовать полную картину поля (`main.go:268–293`).

## Шаг 8. Теоретическое значение — проверка модели

**Физика:** Для **бесконечно длинного** соленоида есть простая формула:

```
B = μ₀ · (N/L) · I
```

Это идеализация. У реального соленоида конечной длины поле в центре чуть меньше из-за **краевых эффектов** — у торцов поле "вытекает" наружу.

**В коде:** `main.go:460–462`

```go
g.bTheory = mu0 * float64(g.N) / g.L * g.I  // теория для бесконечного
cBr, cBz := solenoidField(...)               // численный расчёт центра
g.bCenter = math.Hypot(cBr[0], cBz[0])      // реальное значение модели
```

На экране выводятся оба — можно сравнить насколько реальный соленоид "далёк от идеала".

## Итоговая цепочка

```
Закон Био-Савара
      ↓
Поле 1 витка (эллиптические интегралы K и E)
      ↓
Суперпозиция N витков → полное поле соленоида
      ↓
Тепловая карта (|B| → цвет по шкале plasma)
+ Силовые линии (трассировка методом RK2)
```

## Ключевые вопросы на защиту

**Почему суперпозиция?**
Каждый виток создаёт поле независимо, электромагнитные поля складываются как векторы — это фундаментальное свойство электромагнетизма.

**Почему эллиптические интегралы?**
Это точное аналитическое решение закона Био–Савара для кругового витка в произвольной точке пространства. Альтернатива — численное интегрирование по контуру, но оно медленнее.

**Почему RK2, а не простой шаг Эйлера?**
Метод Эйлера накапливает ошибку — силовые линии "уходят" в сторону. RK2 делает пробный полушаг, уточняет направление и даёт заметно более точную трассировку при той же длине шага.

**Почему поле в центре меньше теоретического?**
Теоретическая формула `B = μ₀nI` выведена для бесконечного соленоида. У реального конечного соленоида поле у краёв ослаблено — витки с одной стороны отсутствуют, вклад не компенсируется.
